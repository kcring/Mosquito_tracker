# data_examples/README.md

## Data provenance

This directory contains two kinds of data:

1. **Synthetic test data** (files prefixed `SYNTHETIC_`) - generated from R code, no real mosquitoes.
2. **Real trajectory data** (`load_real_trajectories.R` + `data/real_trajectories/`) - publicly available
   *Anopheles coluzzii* 3D tracks from an independent lab study (see below).

> **Do not cite synthetic outputs as biological results.**
> The real trajectory data may be cited with the Vielma et al. reference below.

---

## Purpose

This directory provides the minimum synthetic test examples needed to:
1. Validate that each R module runs without errors
2. Demonstrate the pipeline's data structures and output format
3. Generate example plots that illustrate what the analysis would look like with real data

## Synthetic data generation method

Trajectories are generated using the constant-velocity motion model from the paper:

```
r[k+1] = r[k] + rdot[k] * dt + w_r
rdot[k+1] = rdot[k] + w_v
```

where `w ~ N(0, sigma_w)` with `sigma_w = 100e6 mm^2/s^4` (matching `config.m` and paper Table 2).

Initial conditions:
- Position: Normal(0, 200 mm) in x/y; Normal(500, 100 mm) in z  
- Velocity: Normal(0, 600 mm/s) in x/y; Normal(0, 300 mm/s) in z

These values are chosen to produce trajectories that:
- Stay within a realistic swarm volume (~1-2 m across, ~1-2 m height)
- Have speeds in the 0.5-3 m/s range (paper reports 1-4 m/s)
- Are clearly separated enough for tracking

**Measurements** are generated by projecting 3D positions to pixel coordinates using the real calibration values, then adding 2-pixel Gaussian noise.

## How to regenerate

```r
setwd("path/to/mosquito_project")
source("r_reconstruction/run_pipeline.R")
result <- run_synthetic_demo(n_targets = 5, n_frames = 100)
```

Also run the structure test to regenerate calibration and measurement RDS files:

```r
source("r_reconstruction/tests/test_pipeline_structure.R")
```

## What this demonstrates vs what it does NOT demonstrate

| Demonstrates | Does NOT demonstrate |
|---|---|
| Correct camera projection math (km, trm, F) | Real mosquito kinematics |
| Epipolar constraint filtering | Foreground extraction on real images |
| DLT triangulation accuracy | Calibration accuracy |
| Particle filter update structure | Full MHT multi-hypothesis tracking |
| Kalman smoother convergence | Occlusion handling |
| 3D trajectory output format | Biological patterns (swarming, mating) |
| CSV review workflow | GUI-based track editing |
